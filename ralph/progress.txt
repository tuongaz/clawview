## Codebase Patterns
- Run tests with `uv run pytest tests/` (not bare `uv run pytest`) to exclude sniffly reference tests
- Pricing stored as USD per million tokens in `MODEL_PRICING` dict; division by 1_000_000 happens at calculation time
- Cache cost multipliers: creation = input × 1.25, read = input × 0.10
- Fallback to claude-3-5-sonnet-20241022 pricing for unknown models
- Test classes grouped by concern (e.g., `TestGetModelPricing`, `TestCalculateCost`, `TestCacheCosts`, `TestFallback`)
- JSONL messages have `type` field ("user", "assistant", "agent-name", etc.) and nested `message` dict with `content`, `model`, `usage`, `stop_reason`
- Token usage fields: `input_tokens`, `output_tokens`, `cache_creation_input_tokens`, `cache_read_input_tokens`
- Tool errors detected via `is_error: true` in `tool_result` parts of user message content
- Error categorization uses regex patterns from sniffly's `ERROR_PATTERNS` dict (14 categories)
- `_is_real_user_prompt()` filters out tool-result-only user messages and system boilerplate
- Project dir names encode paths: "-Users-tuongaz-dev-foo" → `/Users/tuongaz/dev/foo`
- `compute_project_stats()` in insights.py accepts encoded project dir name, not full path
- "Command" = real user prompt + all subsequent assistant messages until next real user prompt
- Interruptions detected via `[Request interrupted` in message text or `API Error: Request was aborted.` in content
- Cache cost saved formula: `cache_read * 1.0 - (cache_read * 0.10 + cache_created * 1.25)` (normalized to input token cost)
- `command_details` is computed first, then `user_interactions` is derived from it
- Test fixtures for interruptions must use tool_result content (not plain string) to avoid `_is_real_user_prompt()` treating them as commands
- Use `patch.object(os.path, "expanduser", return_value=str(tmp_path))` to test `compute_project_stats()` with fixture data
- Session → project dir: `find_session_file(sid)` → `os.path.basename(os.path.dirname(fpath))` gives encoded project dir name
- `compute_project_stats()` returns plain dicts, serialize with `json.dumps()` (not Pydantic)
- WS endpoints reuse `_load_session_detail()` for activity-based tick speed
- Insights components live in `frontend/src/components/insights/` with barrel export via `index.ts`
- Total project cost = `sum(daily_stats[date].cost for date in daily_stats)` — no top-level cost field exists
- Recharts charts: wrap in `<ChartCard title="...">` → `<ResponsiveContainer>` → chart component; use tree-shaken imports
- Chart color palette: `#667eea` (blue), `#764ba2` (purple), `#48bb78` (green), `#ef4444` (red), `#f59e0b` (orange), `#ed8936` (amber)
- Chart tooltip styling: `contentStyle={{ background: 'var(--bg-primary)', border: '1px solid var(--border)', borderRadius: 6, fontSize: 12 }}`
- `daily_stats[date].cost_breakdown` has `input`, `output`, `cache` (USD) for stacked cost charts; `cost` is the total
- Dynamic interval bucketing: `bucketCommands()` in `dynamicInterval.ts` — shared by all line charts; < 10 days → 4h buckets, >= 10 days → daily, max 60 points. Pass `{ includeInterrupted: true }` for rate charts. `bucketCommandsAndErrors()` merges commands + errors into aligned buckets
- Line charts use cumulative averages (running total ÷ total commands), not per-bucket averages
- Recharts `type="monotone"` provides smooth curves; `dot={false}` + `activeDot={{ r: 4 }}` for clean lines with hover dots
- Dual-axis line charts: use `yAxisId="pct"` (left, `unit="%"`) + `yAxisId="count"` (right, `orientation="right"`) for rate charts combining percentages and counts
